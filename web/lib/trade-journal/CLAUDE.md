# Trade Journal Module

## Overview
Processes DAS Trader CSV exports into round-trip trades and writes them to Google Sheets. Runs on Cloudflare edge runtime â€” no Node.js APIs.

## CSV format (DAS Trader)
Headers: `Event,B/S,Symbol,Shares,Price,Route,Time,Account,Note`
Only rows where `Event === "Execute"` are processed. Everything else (Accept, Cancel, etc.) is filtered out.

## Trade grouping algorithm
Position tracking: Buy = +shares, Sell/Shrt = -shares. When cumulative position returns to 0, that's one complete round-trip trade. The grouper handles multiple partial fills and computes volume-weighted average entry/exit prices.

## Google Sheets API (edge-compatible)
No `googleapis` SDK. All calls use `fetch` directly against `https://sheets.googleapis.com/v4/spreadsheets`.

Auth flow:
1. Parse service account JSON from env var
2. Build JWT with `iss`, `scope`, `aud`, `iat`, `exp`
3. Sign with Web Crypto API (`RSASSA-PKCS1-v1_5` / `SHA-256`)
4. Exchange JWT for access token at `https://oauth2.googleapis.com/token`

Key functions:
- `getAccessToken()` â€” JWT-based OAuth2 service account auth
- `ensureSheetTab()` â€” finds or creates a tab for the account, applies formatting
- `appendTrades()` â€” main entry point: dedup, append, compute stats
- `populateInstructionsSheet()` â€” one-shot: writes column reference to Instructions tab

## Column layout (50 columns, A-AX)
Auto-filled from CSV: Date, Entry Time, Exit Time, Duration, Symbol, Side, Shares, Avg Entry, Avg Exit, # Partials, P&L.
Formula columns: Stop (Entry Â± R/Shares), P&L(R) (P&L/R), 1R-6R (Y/N whether Max R Before Stop reached each R-multiple).
Max R Before Stop: Order-aware enrichment â€” walks 1-min bars from entry to 16:00 ET, tracks max favorable R-multiple, stops if stop-loss hit. Requires R filled. Farthest Price is the stock price at that point.
Per-trade manual: R (Risk), Setup (dropdown), Process Followed? (dropdown), Notes, Conviction 1-3 (dropdown), Catalyst (multi-select dropdown: Earnings, Upgrade/Downgrade, FDA/Regulatory, Sector Momentum, Gap Only, Key Daily Level, Day 2, Pullback to DEMA, Other).
Daily manual (fill once on first trade of the day): Sleep Score (0-100), Readiness Score (0-100), Emotional State (dropdown), Market Bias (dropdown).
Market data enrichment (auto from Polygon): #1m, #5m, #1H, %Gap, %ATR, RVOL, %VWAP, OR Size ($), OR %ATR, OR High, OR Low, Max R Before Stop, Farthest Price, Breakout Vol Ratio, Prior Close Loc, Dist 20 SMA (%), Dist 50 SMA (%), Float, Avg $ Vol, SPY Dir, VIX.
All formulas are generated by `buildFormulas()` and used by both `tradeToRow()` (new trades) and `migrateTabIfNeeded()` (existing rows).

## Dedup
Key: `Date|Symbol|normalizedEntryTime|Side`. Times are normalized (leading zeros stripped) because Google Sheets drops them (e.g., `09:30:46` â†’ `9:30:46`).

## Known gotchas
- `Infinity` â†’ use `9999` sentinel (JSON.stringify turns Infinity into null)
- Time normalization is critical for dedup â€” always strip leading zeros before comparison
- Google Sheets currency columns return values like `$32.21` â€” strip `$` and `,` when parsing P&L values back
